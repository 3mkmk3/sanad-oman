# سند عمان — دليل النشر على Vercel 🚀

## محتويات الحزمة

| الملف | وظيفته |
|---|---|
| `index.html` | التطبيق كامل (التصميم الجديد v2) |
| `api/plan.js` | دالة خادمة تتصل بـ Claude API بأمان — مفتاحك لا يظهر أبداً للزوار |
| `api/submit.js` | تستقبل طلبات تسجيل الأنشطة التجارية وتحفظها في قاعدة بيانات حقيقية |
| `api/requests.js` | تعيد كل الطلبات — للمشرف فقط، محمية بكلمة مرور من الخادم |
| `api/review.js` | تنفّذ القبول/الرفض — للمشرف فقط |
| `api/places.js` | تعيد الأنشطة المعتمدة ليراها كل الزوار |
| `api/_kv.js` / `api/_auth.js` | ملفات مساعدة داخلية (تخزين + تحقق مشرف) |
| `manifest.json` | ملف تعريف PWA (الاسم، الألوان، الأيقونات) |
| `sw.js` | Service Worker — يخلي التطبيق يشتغل حتى بدون إنترنت |
| `icon-192.png` / `icon-512.png` | أيقونات التطبيق عند التثبيت على الشاشة الرئيسية |

## خطوات النشر (10 دقائق تقريباً)

### 1. احصل على مفتاح Anthropic API
- ادخل على console.anthropic.com وسجّل حساب
- من قسم **API Keys** أنشئ مفتاحاً جديداً وانسخه (يبدأ بـ `sk-ant-`)
- ملاحظة: استخدام API مدفوع حسب الاستهلاك، لكن طلبات الجدول السياحي قصيرة ورخيصة جداً

### 2. ارفع الملفات على GitHub
- أنشئ حساباً على github.com إن ما عندك
- **New repository** → سمّه مثلاً `sanad-oman` → اجعله Private إذا تحب
- اضغط **uploading an existing file** وارفع كل ملفات هذه الحزمة **بنفس ترتيب المجلدات** (ملف `plan.js` لازم يكون داخل مجلد `api`)

### 3. اربط المشروع بـ Vercel
- ادخل على vercel.com وسجّل بحساب GitHub نفسه
- **Add New → Project** → اختر مستودع `sanad-oman` → **Import**
- لا تغيّر أي إعدادات (Vercel يتعرف على الملفات تلقائياً) → **Deploy**

### 4. أضف قاعدة بيانات مجانية لطلبات التسجيل (Upstash Redis)
- من داخل مشروعك على Vercel: تبويب **Storage** → **Create Database** → اختر **Upstash — Redis** (فيه خطة مجانية كافية جداً لهذا الاستخدام)
- بعد الإنشاء، اربطه بمشروعك (**Connect Project**) — سيضيف Vercel تلقائياً متغيّرين للبيئة: `KV_REST_API_URL` و `KV_REST_API_TOKEN`

### 5. أضف المفاتيح السرية
- **Settings → Environment Variables** وأضف:
  - `ANTHROPIC_API_KEY` — مفتاح Claude الذي نسخته
  - `ADMIN_PASSWORD` — كلمة مرور من اختيارك للوحة التحكم (لن تظهر أبداً في كود الصفحة، الآن يتم التحقق منها على الخادم فقط)
- إذا لم يضِفهما Vercel تلقائياً من الخطوة السابقة، تأكد أن `KV_REST_API_URL` و `KV_REST_API_TOKEN` موجودان أيضاً
- احفظ، ثم من تبويب **Deployments** اضغط على آخر نشر → **Redeploy**

### 6. اختبر! ✅
- افتح الرابط `https://sanad-oman.vercel.app` (أو ما يشبهه)
- جرّب ميزة "جدولي السياحي" — إذا بنى الجدول، فكل شيء يعمل
- جرّب "تسجيل نشاطك التجاري" ثم افتح "لوحة التحكم" وسجّل الدخول بكلمة `ADMIN_PASSWORD` التي وضعتها — يفترض أن ترى الطلب الذي أرسلته
- من الجوال: افتح الرابط → قائمة المتصفح → **إضافة إلى الشاشة الرئيسية** — سيُثبَّت كتطبيق حقيقي بأيقونته

## للتجربة محلياً (اختياري)

```bash
npm i -g vercel
cd sanad-oman
vercel dev
```
`vercel dev` يشغّل الدوال الخادمة محلياً أيضاً. فتح `index.html` مباشرة من الجهاز سيُشغّل كل شيء **ما عدا** ميزة الذكاء الاصطناعي (تحتاج الخادم).

## ملاحظات أمان تم حلّها ✅

- كلمة مرور المشرف الآن تُتحقق منها فقط على الخادم (`api/_auth.js`) — لا تظهر أبداً في كود الصفحة.
- طلبات التسجيل تُحفظ الآن في قاعدة بيانات حقيقية (Upstash Redis) وتصل لك كمشرف من أي جهاز، وليس فقط متصفح مقدّم الطلب.
- بعد الموافقة على طلب، يظهر النشاط لكل الزوار فوراً (عبر `api/places.js`) وليس فقط في متصفحك.
